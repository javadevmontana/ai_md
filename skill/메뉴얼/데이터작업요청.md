# 공문요청 범위 가이드

고객사 데이터 관련 작업 시 공문요청 필요 여부를 판단하는 기준이다.

## 공문요청 필요

다음 작업은 반드시 고객사 공문을 받은 후 진행한다:

| 구분              | 예시                                                |
| ----------------- | --------------------------------------------------- |
| 개인 데이터 요청  | 메일 내용, 개인정보가 포함된 데이터 추출            |
| 마이그레이션 관련 | 데이터 구조 파악, 데이터량 산정, 이관용 데이터 추출 |
| DB 데이터 삭제    | 레코드 삭제, 테이블 데이터 정리                     |
| DB 데이터 수정    | 고객 요청에 의한 데이터 변경                        |

### 예외: 공문 없이 진행 가능

- **오류 수정**: 시스템 오류로 인해 잘못 저장된 데이터 정정

## 공문요청 불필요

다음 작업은 공문 없이 진행 가능하다:

| 구분            | 예시                                    |
| --------------- | --------------------------------------- |
| 통계성 자료     | 특정 제목의 결재 건수, 부서별 처리 건수 |
| 집계 데이터     | 월별/분기별 사용량 통계, 현황 리포트    |
| 익명화된 데이터 | 개인 식별 불가능한 집계 정보            |

## 판단 기준 요약

```
개인정보 포함? ─────────────── Yes → 공문 필요
데이터 삭제/수정? ──────────── Yes → 공문 필요 (단, 오류 수정 제외)
마이그레이션 목적? ─────────── Yes → 공문 필요
통계/집계만 필요? ──────────── Yes → 공문 불필요
```

## 공문요청이 필요한 내용의 경우 아래와 같은 메일 템플릿으로 안내해줘

[고객사 요청사항]
예시)2019~2021년에 작성된 결재 문서 중 제목에 '출장신청서' 라는 단어가 들어가는 기록물철 문서의 개수 같은 세부 내용

1. 데이터 추출 요청 건
   : 개인정보 추출은 개인정보보호법 위반이 될수있는 민감한 작업입니다. 공문은 위험으로부터 회사와 담당자를 보호하는 최소한의 안전장치입니다
   만약 직원의 동의 또는 회사 사규/규정에 열람, 취득의 내용이 있으면 해당 내용을 근거로 요청하시면 됩니다.
   아래와 같은 내용을 작성하셔서 공문을 보내주시면 검토 후 데이터 전달드리도록 하겠습니다.

- 공문 발송시 포함 내용

1.  Data의 소유자
2.  추출할 Data를 식별할 수 있는 정보 전체 (제목, 생산일 등)
3.  추출 요청자
4.  추출 요청하는 주요한 이유
5.  취업규칙/사규 또는 근로자 동의된 경우는 해당 내용 추가 삽입 (Option)
    - 아래 추출 가능 조건의 1), 2)번 조항
6.  직인 (기본은 회사 대표직인, 또는 감사/준법팀의 팀장 직인)

- 임의 추출시 가능 조건

1.  회사 규정/취업규칙에 업무상 생산물 (메일/Document) 등의 모니터링 범위와 절차가 명확히 기재
2.  근로자가 사전에 이에 대해 명시적으로 동의
3.  업무상 매우 중요한 사유(시스템 장애 대응, 법적 의무 등)
4.  내부고발 신분을 특정할 목적이 아니어야 함

- 근로자 동의 없이 Data 추출의 문제

1.  근로자 개인정보 보호법/통신비밀보호법 위반
2.  공익신고자 보호법 위반
3.  노동자 인권 보호 내용

## 데이터변경이나 추가는 건단위 요청이라면 공문까지 요청하진 않고 있어 아래와 같은 메일 템플릿으로 안내해줘

작업내용
: 예)대학혁신지원사업단 김웅배선생님의 완결문서/결재완료함에 삭제된 공문을 재생성요청

작업사유
: 사용자의 잘못된 삭제로 인한 재생성 요청
=> 완결문서/결재완료함 리스트에 삭제된 공문을 재생성

요청자
: 전산정보팀 담당 윤운일

요청으로 인한 데이터 변경 및 삭제는 담당자의 신의성실 원칙에 따라 수정 작업 중 예기치 않은 문제 발생 시 책임을 묻지 않을 것을 약속합니다.

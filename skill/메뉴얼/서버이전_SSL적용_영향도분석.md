# 서버이전 및 SSL 적용에 따른 그룹웨어·서드파티 솔루션 영향도 분석

---

## 1. 서버이전

### 1-1. 볼륨

- 재부팅 과정에서 NAS 등 마운트 볼륨이 해제될 수 있으므로 **기존 마운트 정보를 사전에 확인**할 것

### 1-2. 네트워크

서드파티 솔루션을 포함하여 **필수 인바운드 & 아웃바운드 포트 정책**을 사전에 전달해야 한다.

| 구분 | 포트 | 비고 |
|------|------|------|
| 그룹웨어 웹 | 80, 443 등 | WAS 내 웹포트 및 `xclick-config.xml` 내 IP/Port |
| 그룹웨어 모바일 | 80, 443 등 | WAS 내 웹포트 및 `xclick-config.xml` 내 IP/Port |
| 메신저 | 80, 443, 15221~15231 | 총 13개 포트 |
| 서드파티 메일 | 25, 465, 110, 995, 80, 443 | SMTP/POP3/HTTP(S) |

### 1-3. 물리적 서버이전으로 IP가 변경될 때

- `xclick-config.xml` 내 **domain** 부분이 IP로 되어 있다면 수정 필요
- `xclick-config.xml` 내 **server IP** 변경 필요
- 고객사에 **DNS 등 도메인에 매핑된 IP 변경 요청** 필수

### 1-4. 가상화 서버 또는 클라우드 서버로 이전될 때

| 체크 항목 | 설명 |
|-----------|------|
| 퀄리티아 메일 엔진 | MAC ADDRESS 변경 시 **라이선스 재발급 필수** (사전 체크) |
| 그룹웨어 | hostname 변경 시 **라이선스 재발급 필수** (사전 체크) |
| 환경 셋팅 | 신규 서버에 모든 서비스를 새로 환경 셋팅 및 데이터 이전 필요 → **서버이전 체크리스트 템플릿 참고** |

> ⚠️ **주의:** OS에서 hostname으로 확인하면 다른 값이 나올 수 있으므로 반드시 체크할 것

## 2. SSL 적용

### 2-1. 인증서 발급 요청

- 고객사에 인증서 발급 요청
- 서드파티 솔루션이 **다른 1단계 도메인**을 사용할 수 있으므로 사전 체크
- `*.gw.com` 형태의 **와일드카드 인증서 발급 권장**

### 2-2. iframe 기간계 시스템 확인

- 기능상 **iframe으로 기간계 시스템을 표시하는지 확인** 필수
- HTTPS 상태에서 HTTP 호출은 **불가능** (Mixed Content 차단)
- HTTP 상태에서 HTTPS 호출은 **가능**
- 사전에 영향도를 파악하고 **고객사에 반드시 가이드**해야 함

### 2-3. WAS별 인증서 적용 주체

| WAS 종류 | 적용 주체 | 비고 |
|----------|-----------|------|
| Tomcat | **비즈웰 직접 적용 및 갱신** | 웹한글기안기 등 서드파티는 제외 |
| Jeus, Resin 등 상용 WAS | **고객사를 통해 적용 및 갱신** | - |

### 2-4. 서드파티 SSL 적용

| 서드파티 | SSL 적용 비용 | 비고 |
|----------|--------------|------|
| 문서변환 - SAT | **무료** (인증서만 전달) | - |
| 문서변환 - 사이냅 | **별도 비용 확인 필요** | - |
| 메일 - 퀄리티아 (포스티안) | **무료** | 영업시간 내 처리 (09~18시) |
| 메신저 | **체크 대상 아님** | - |

---

## 3. 메신저

- 비즈웰 Buzz 메신저의 경우 **기본적으로 SSL을 적용하지 않음**
- 고객사 별도 요청 시에만 진행 → **연구소 공수 체크 필요**

---

## 4. 모바일

- 기본적으로 SSL **포함하여 적용**
- DMZ망으로 별도 서비스가 분리되어 있거나 Proxy로 연결된 경우, **해당 시스템에도 적용 필요**
- 기본 프로토콜 변경(HTTP → HTTPS)으로 **앱 재빌드 필요** 안내
  - 자동 업그레이드 가능하나, 고객사에서 배포 불편을 이유로 거절하는 경우가 있으므로 **적용 여부 사전 확인**

---

## 5. 기타 주의사항

- **대용량 메일 링크 URL 변경:** 과거 발송된 HTTP URL은 SSL 적용 후 그룹웨어 내에서 접근 불가 (보안 위배). 새 창으로 오픈만 가능
- **인증서 로컬 테스트:** 발급된 인증서는 개발자 hosts 파일을 수정하여 로컬에 직접 적용해 보는 것을 권장
- **서드파티 영향도:** 서버이전이나 SSL 등 서버 환경 변화가 있을 때 **영향도 및 비용에 대해 신중히 검토**할 것

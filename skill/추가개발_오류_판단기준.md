# 추가개발 vs 오류 판단 기준

## 개요

그룹웨어 유지보수 시 **추가개발(유상 지원)**과 **오류(무상 지원)**를 구분하기 위한 판단 기준입니다.

---

## 추가개발 (유상 지원)

공수가 들어가는 작업으로, 별도 비용이 발생합니다.

### 해당 사례

| 구분 | 설명 |
|------|------|
| 신규 기능 추가 | 현재 그룹웨어에 없는 기능을 추가하는 경우 |
| 기능 수정/확장 | 이미 있는 기능을 수정하거나 덧붙이는 경우 |
| 기능 개선 | 이미 있는 기능이나 사용상 불편한 경우 기능 개선 |
| 환경 변화 대응 (보안) | 보안취약점이지만 OS나 브라우저 등 환경적인 변화에 따라 수정해야 하는 경우 |
| 보안 강화 | 보안취약점이지만 2차 인증 등 보안강화에 해당되는 경우 |
| 환경 변화 대응 (일반) | 잘 되던 기능이 고객사의 네트워크나 브라우저 등 환경변화에 따라 수정해야 하는 경우 |
| 데이터 추출 | 통계성 데이터 추출 |

### 핵심 판단 기준

> **"제품 자체의 결함이 아닌, 요구사항 추가 또는 외부 환경 변화에 따른 대응인가?"**

---

## 오류 (무상 지원)

제품 자체의 결함으로 인해 정상 동작하지 않는 경우로, 무상으로 지원합니다.

### 해당 사례

| 구분 | 설명 |
|------|------|
| 데이터 오류 | 사용자의 의도와 다르게 데이터가 다르게 보이거나 처리된 현상 |
| 기능 장애 | 에러가 발생되어 원하는 기능 동작이 되지 않을 때 |
| 심각한 보안 취약점 | 보안취약점 중 매우 심각하여 데이터 탈취가 손쉽게 가능한 경우 |
| 성능 결함 | 쿼리 속도 지연 튜닝 |

### 핵심 판단 기준

> **"동일 환경에서 제품이 설계된 대로 동작하지 않는가?"**

---

## 경계선 판단 가이드

| 상황 | 분류 | 이유 |
|------|------|------|
| 버그로 인한 데이터 오류 | 오류 | 제품 결함 |
| 고객이 원하는 형태로 데이터 표시 변경 | 추가개발 | 요구사항 변경 |
| SQL Injection 취약점 | 오류 | 심각한 보안 결함 |
| 2차 인증 추가 요청 | 추가개발 | 보안 강화 (신규 기능) |
| Chrome 업데이트 후 동작 불가 | 추가개발 | 외부 환경 변화 대응 |
| 동일 환경에서 갑자기 에러 발생 | 오류 | 제품 결함 |
| 외부 서비스 정책 변경 대응 (OAuth 등) | 추가개발 | 외부 환경 변화 대응 |
| 월별 사용 현황 데이터 추출 요청 | 추가개발 | 통계성 데이터 추출 |

---

## 요약

| 구분 | 핵심 키워드 | 비용 |
|------|-------------|------|
| 추가개발 | 신규, 수정, 개선, 환경변화, 보안강화, 데이터추출 | 유상 |
| 오류 | 버그, 에러, 심각한 보안결함, 성능결함 | 무상 |
